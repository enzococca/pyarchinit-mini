# v2.1.61 Bundled Release — Design Spec

- **Date:** 2026-05-13
- **Target version:** pyarchinit-mini v2.1.61
- **Author:** Enzo Cocca (brainstormed via Claude)
- **Status:** Approved for planning

## 1. Summary

Single release containing 8 changes in 4 groups:

- **Group A — Pottery completion (4 items):** restore action buttons in list rows; integrate media upload/preview in form & detail; populate `THESAURUS_MAPPINGS` with archaeological vocabulary for pottery fields; add sidebar entries.
- **Group B — AI (2 items):** new `AppSetting` model with Fernet-encrypted values backing an admin UI to set OpenAI/Claude API keys at runtime; add `gpt-5.5` and `claude-sonnet-4-7` to the supported-model lists.
- **Group C — UI fix (1 item):** force visible horizontal scrollbar on `.table-responsive` containers and pin table headers.
- **Group D — Backup (1 item):** admin backup module with manual trigger, scheduled auto-backup (daily/weekly/monthly), file listing with download/delete, retention rotation.

## 2. Goals

- Pottery becomes feature-complete (parity with `inventario_materiali`).
- Admins can configure AI keys from the UI without editing env vars.
- Multimedia table is usable on all viewport widths.
- Admins can take/schedule database backups without shell access.

## 3. Non-Goals

- Per-user AI keys — single instance-wide key set.
- Backup file uploaded to cloud storage — local-disk only.
- Backup encryption at rest — files are dropped under
  `~/.pyarchinit_mini/backups/` with normal filesystem permissions.
- Multimedia card-layout responsive rewrite — only CSS scrollbar + sticky
  header.
- Bibliography sub-tab in pottery — still placeholder.

## 5. Group B — AI

### B1. AppSettings + AI keys admin

#### Model

`pyarchinit_mini/models/app_setting.py`:

```python
class AppSetting(BaseModel):
    __tablename__ = 'app_settings'

    id_setting = Column(Integer, primary_key=True, autoincrement=True)
    key = Column(String(100), unique=True, nullable=False, index=True)
    value = Column(Text)            # Encrypted if `is_secret` is True
    is_secret = Column(Boolean, default=False, nullable=False)
    description = Column(Text)
```

Inherits the standard BaseModel sync columns.

#### Service

`pyarchinit_mini/services/app_setting_service.py`:

- `AppSettingService.get(key) -> Optional[str]` — returns decrypted value (or raw if not secret)
- `AppSettingService.set(key, value, is_secret=False, description=None) -> AppSetting`
- `AppSettingService.list_settings() -> List[AppSetting]`
- `AppSettingService.delete(key) -> bool`

Encryption uses `cryptography.fernet.Fernet` with a master key stored at
`~/.pyarchinit_mini/secret.key`. The key file is auto-generated on first
service instantiation with `os.chmod(path, 0o600)`. If `cryptography` is not
installed, fall back to plain-text storage with a logged warning (the
package is added to `pyproject.toml` so this only happens during dev).

#### Admin route + UI

New route in `app.py`:
- `GET  /admin/settings/ai` — render form with current keys masked (e.g.
  `sk-...****` show last 4 chars).
- `POST /admin/settings/ai` — accept `openai_api_key`, `anthropic_api_key`,
  `default_provider`, `default_model`. Save via `AppSettingService.set(...)`.
  Flash success.

Template `templates/admin/settings_ai.html` with 4 fields. Sensitive fields
have `<input type="password">` and a show/hide eye toggle. Submit button.

Admin-only via the existing `@admin_required` decorator from
`pyarchinit_mini/web_interface/auth_routes.py:87`.

#### AIService integration

Modify `pyarchinit_mini/services/ai_assistant_service.py` `__init__`:

```python
def __init__(self):
    # Priority: AppSetting → env var → default
    try:
        from .app_setting_service import AppSettingService
        from ..database.connection import get_db
        svc = AppSettingService(get_db())
        self.provider = svc.get("ai_provider") or os.environ.get("AI_PROVIDER", "openai")
        self.api_key = svc.get(f"{self.provider}_api_key") or os.environ.get("AI_API_KEY", "")
        self.model = svc.get("ai_model") or os.environ.get("AI_MODEL", DEFAULT_MODELS[self.provider])
    except Exception:
        # Fallback to env-only if AppSetting infra not ready (e.g. during migrations)
        self.provider = os.environ.get("AI_PROVIDER", "openai").lower()
        self.api_key = os.environ.get("AI_API_KEY", "")
        self.model = os.environ.get("AI_MODEL", DEFAULT_MODELS.get(self.provider, "gpt-5.4-mini"))
```

## 7. Group D — Backup admin

### D1. BackupService

`pyarchinit_mini/services/backup_service.py` (new). Wraps existing
`ImportExportService._create_backup` for the actual dump work:

- `BackupService.create_backup_now(db_url=None) -> Dict[str, Any]` — runs
  backup synchronously, returns `{ path, size_bytes, format, created_at }`.
- `BackupService.list_backups() -> List[Dict[str, Any]]` — scans
  `~/.pyarchinit_mini/backups/`, returns sorted-by-mtime descending.
- `BackupService.delete_backup(filename) -> bool` — removes file safely (path
  must remain inside backups dir).
- `BackupService.get_schedule() -> Dict[str, Any]` — reads from `AppSetting`:
  `backup_frequency`, `backup_keep_last`, `backup_enabled`.
- `BackupService.set_schedule(frequency, keep_last, enabled) -> None`.
- `BackupService._enforce_retention()` — after a backup, prune to keep last
  `keep_last` files (default 7).

Backup format auto-detected:
- Postgres → `pg_dump -F c` (custom format) if available, else JSON snapshot.
- SQLite → file copy.

Files named `pyarchinit_backup_YYYYMMDD_HHMMSS.<ext>`.

### D2. Background scheduler

Lightweight in-process thread (no APScheduler dep). On `create_app()`:

```python
_backup_thread = threading.Thread(target=_backup_scheduler_loop, args=(app,), daemon=True)
_backup_thread.start()
```

`_backup_scheduler_loop`:

```python
def _backup_scheduler_loop(app):
    while True:
        try:
            with app.app_context():
                bs = BackupService(app.db_manager)
                sched = bs.get_schedule()
                if sched["enabled"]:
                    if _due_for_backup(sched):
                        bs.create_backup_now()
                        bs._enforce_retention()
        except Exception as e:
            logger.error(f"backup_scheduler_loop error: {e}")
        time.sleep(3600)  # check every hour
```

`_due_for_backup(sched)`: compares last backup mtime to current time
against frequency (daily=24h, weekly=7d, monthly=30d).

### D3. Admin routes + UI

- `GET  /admin/backups` — render dashboard: list existing backups, current
  schedule, Backup Now button, Schedule form.
- `POST /admin/backups/now` — trigger immediate backup, flash result.
- `POST /admin/backups/schedule` — save schedule.
- `POST /admin/backups/<filename>/delete` — delete file.
- `GET  /admin/backups/<filename>/download` — `send_file` from backups dir.

Template `templates/admin/backups.html` with:
- Card: "Backup Now" button + last backup info
- Card: schedule form (radio: disabled/daily/weekly/monthly + keep_last number)
- Table: existing backups (filename, size, created_at, actions: download/delete)

Admin-only via `@admin_required` (same auth as `/admin/settings/ai`).

## 10. Migration

- New `app_settings` table created automatically by
  `Base.metadata.create_all()` on first boot after upgrade.
- New `backups/` directory under `~/.pyarchinit_mini/` created on first
  `BackupService.list_backups()` call.
- First-boot upgrade path runs without user action — existing AI users
  continue to use env vars until they switch via the new admin UI.
